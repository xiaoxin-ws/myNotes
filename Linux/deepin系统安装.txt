工具：两个U盘，deepin系统镜像，微pe安装软件，win10镜像（备用或者装系统）

1.刻录启动盘（deepin的刻录工具刻录deepin启动盘，微pe刻录pe工具盘）下面如果用的着pe的话重启按住esc选择pe盘即可
2.保证安装盘为grup列表引导（如不是请用微pe（或者其他）里面的DiskGenius软件将要安装系统的那块硬盘转化为grup格式（这会貌似影响磁盘的数据请做好备份，保证有一个esp分区，双系统的话，还要保证两个分区建议至少60g）），分配物理磁盘空间不小于60g
3.双系统建议先装windows，可用上面的微pe装。装好后建议启动windows做好配置工作才装deepin（单系统跳过这步）
4.重启按住esc键选择刚才制作的且带有UEFFI字样的启动盘(双系统不选这个的话另一个系统不会出现在deepin的系统选择界面)
5.选择install deepin
6.进入安装，选择上面分配那个空分区，一步步安装即可（最后安装好后拔掉启动盘重启进入系统）
7.启动后显卡管理程序 换为核心显卡，重启，卡死就按住电源键强制重启（下面两个显卡目前都有重启，睡眠，待机关机就卡死的问题，第一个核显，重启卡死，其他都正常，将就着用）找到控制面板，设置睡眠和待机都为永久（防止睡眠卡死，下面永久配置后解决这个问题）
8.如果这时启动卡logo的话，按住电源键强制关机，然后开机到deepin系统选择界面时，马上按e进入grup编辑，找到linux /boot/vmlinuz-xxx-deepinx-amd64 root=UUID=57d9aa6c-2452-4374-b4b8-bbd81a2975c2 ro splash quiet，在 ro splash quiet 的后面添加下面代码，注意空格隔开（零时配置，）
------------------------------------------------------
acpi_osi=! acpi="windows 2009"
------------------------------------------------------
随后按f10保存启动，进入系统后，
找到控制面板，设置睡眠和待机都为永久（防止睡眠卡死，下面永久配置后解决这个问题）
提示显卡转换完毕是否重启，选否，不重启
打开终端（永久配置）
---------------------------------
sudo gedit /boot/grub/grub.cfg
---------------------------------
注意在gedit后面有一个空格，（如果在输入之后显示找不到指令，则先需要安装gedit）
---------------------------------
sudo apt-get install gedit
---------------------------------
在弹出的文本页面里，再次找到linux /boot/vmlinuz-xxx-deepinx-amd64 root=UUID=57d9aa6c-2452-4374-b4b8-bbd81a2975c2 ro splash quiet
添加下面代码,理论上加在第一个就行了，但是保险起见三处都加上
----------------------------------
acpi_osi=! acpi="windows 2009"
---------------------------------
保存退出。

使用Deepin（或Ubuntu）和Windows 10双系统在登陆进入后卡在logo或黑屏的，如果用双显卡导致的，请按下面解决方法处理。


        说明

        在笔记本上有I卡与N卡双显卡，在安装Deepin完成后要进入系统却一直卡在logo处，无法正常进入系统，针对这个问题，下面介绍四种解决方法，对Deepin（基于Ubuntu的Linux发行版本）或是Ubuntu有效。


        解决方法一

        具体的操作步骤如下：

        1.在开机进入选择系统的界面的时候不要进行选择，而是按e进入引导文本编辑页面。

        2.找到下面的这一行：

        linux /boot/vmlinuz-xxx-deepinx-amd64 root=UUID=57d9aa6c-2452-4374-b4b8-bbd81a2975c2

        ro splash quiet

        在quiet的后面空一格然后加上acpi_osi=! acpi="windows 2009"后按F10保存进入Deepin系统。

        3.然后在Deepin里面打开终端，在里面输入sudo su和sudo gedit /boot/grub/grub.cfg（注意在gedit后面有一个空格）这两条语句（如果在输入之后显示找不到指令，则先需要安装gedit，安装的语句是sudo apt-get install gedit）在弹出的文本页面里，再次找到linux /boot/vmlinuz-xxx-deepinx-amd64 root=UUID=57d9aa6c-2452-4374-b4b8-bbd81a2975c2ro splash quiet然后跟上面在quiet后面空一格之后输入acpi_osi=! acpi="windows 2009"，之后保存，关闭。

        4.在终端里面输入sudo gedit /etc/default/grub然后找到开头是GRUB_CMDLINE_LINUX_DEFAULT=在它的等号后面的语句改成splash quiet acpi_osi=! acpi="windows 2009"，然后按Esc，冒号，输入wq保存。


        解决方法二

        具体操作步骤如下：

        1.和方法1一样在在开机进入选择系统的界面按e，然后也是找到带有splash quiet的那一行，将splash quiet删除后输入nomodeset，按F10保存进入。（如果进去之后一直待在命令行的，就可以跳过该方法了。）

        2.打开终端，在里面输入sudo gedit /etc/default/grub，找到GRUB_CMDLINE_LINUX_DEFAULT=这一行，在它的等号后面的语句改成"quiet splash nomodeset"，然后按Esc，冒号，输入wq保存。


        解决方法三

        具体步骤如下：

        1.一样在进入时按e，然后也是找到带有splash quiet的那一行，在它们的后面加上acpi=off,F10保存退出。

        2.进入Deepin系统之后的操作步骤就是和方法1大致一样的了，就只是将方法1中的acpi_osi=! acpi="windows 2009"改成acpi=off即可。


        解决方法四

        具体步骤如下：

        1.在bios中设置关闭独立显卡，只要集成显卡。

        2.在进入系统之前将自己制作好的upan插在主板的USB 3.0接口上就可以了。
